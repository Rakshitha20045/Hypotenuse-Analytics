# Grok API Models

Grok is xAI’s model family for chat, reasoning, coding, and multimodal assistant use. It is designed to be fast, conversational, and capable for both consumer and developer workflows.

## Big picture
Grok models are used for:
- General chat and Q&A.
- Reasoning and coding.
- Fast assistant workflows.
- Multimodal understanding in supported versions.
- Social/web-aware assistant experiences.

## Core model families

### Grok-4
The flagship Grok model. Best for:
- Complex reasoning.
- Coding.
- Agentic workflows.
- High-quality assistant behavior.

### Grok-4 Fast
A speed-optimized version of the flagship model. Best for:
- Lower-latency chat.
- Interactive assistants.
- Production use where speed matters.

### Grok-3 and lighter variants
Older or more efficient Grok options. Best for:
- General chat.
- Simple coding help.
- High-volume or cost-sensitive tasks.

### Multimodal Grok variants
Used when image or screenshot understanding is supported. Best for:
- UI analysis.
- Visual Q&A.
- Image-based reasoning.

## Best use cases
- Chat assistants.
- Coding help and debugging.
- Agentic workflows.
- Summarization and rewriting.
- Visual understanding.
- Research-style assistants.

## Model selection
| Need | Best choice |
|---|---|
| Hard reasoning | Grok-4 |
| Fast responses | Grok-4 Fast |
| General chat | Grok-4 or Grok-3 |
| Coding help | Grok-4 |
| Lower cost | Grok-3 Mini or Fast variants |
| Image understanding | Multimodal Grok variant |

## Practical guidance
- Use flagship models for complex work.
- Use fast variants for real-time experiences.
- Use lighter variants for cost-sensitive tasks.
- Use the API when building products.
- Use the consumer product for quick testing.
